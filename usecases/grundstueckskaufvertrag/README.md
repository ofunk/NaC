# Real-estate Purchase Contract

Status: starter

## Goal

Prepare a notary-office usecase package for a real-estate purchase contract.
The package will connect land-register readiness, parties, approvals, draft
contract preparation, execution tracking, and evidence metadata.

## Boundaries

- No final legal advice is generated by the LLM.
- No protected portal is scraped or automated without a reviewed connector.
- No real personal data, land-register excerpts, or secrets are stored in Git.

## Initial Dependencies

| Layer | Dependency | Purpose |
| --- | --- | --- |
| Plugin | `noc-regulated-core` | Shared regulated workflow guardrails. |
| Plugin | `noc-grundbuch-portal` | Land-register access and evidence companion. |
| Plugin | `noc-bnotk-xnp` | Notary workstation readiness when needed. |
| Workflow | `workflows/contracts/` | Intake, approval, and evidence contract. |
| Workflow | `workflows/python/` | Deterministic checks and plan preview. |

## Delivery Plan

1. Define intake contract for property, parties, purchase price, financing,
   encumbrances, and approvals.
2. Bind land-register evidence requirements to plugin outputs.
3. Create draft-preparation and approval workflow contract.
4. Define evidence metadata for draft, review, appointment, execution, and Day2
   follow-up.
