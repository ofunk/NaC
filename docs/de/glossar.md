# Glossar Fuer Nicht-Technische Leser

Dieses Glossar erklaert NoC-Begriffe so, wie sie fuer einen Notariatsbetrieb
relevant sind.

| Begriff | Bedeutung Fuer Das Buero |
| --- | --- |
| Git | Das versionierte Protokoll. Es zeigt, wer wann welche Regel, Vorlage oder Entscheidung geaendert hat. |
| Repository | Der gemeinsame Arbeitsordner mit Regeln, Usecases, Workflows, Tests und Nachweisen. |
| Privater Fork | Die eigene, nicht oeffentliche Kopie fuer ein konkretes Buero oder Pilotprojekt. Dort werden lokale Rollen und Betriebsregeln gepflegt. |
| Pull Request | Ein gepruefter Aenderungsvorschlag. Er ersetzt ungepruefte Direktbearbeitung an wichtigen Regeln. |
| Review | Die fachliche oder technische Pruefung, bevor eine Aenderung verbindlich wird. |
| Merge nach `main` | Die Aenderung wird in den verbindlichen Hauptstand uebernommen. |
| Knowledge Graph / KG | Die strukturierte Liste offener Angaben, Dokumente, Entscheidungen, Gates und Nachweise fuer einen Vorgang. |
| Usecase | Eine konkrete notarielle Vorgangsart, zum Beispiel Immobilienkaufvertrag oder GmbH-Gruendung. |
| Workflow | Ein wiederholbarer Ablauf aus Eingabe, Pruefung, Freigabe und Nachweis. |
| Plugin | Ein Begleitpaket fuer eine bestimmte technische oder fachliche Faehigkeit, zum Beispiel Kartenleser-Readiness oder Registerpfad. |
| Quality Gate | Ein automatischer Prueflauf. Er zeigt, ob Tests, Datenschutzregeln, Sprachregeln und KG-Regeln eingehalten werden. |
| SBOM / AI-SBOM | Eine Lieferkettenliste fuer Software, Modelle, externe Dienste und lokale Komponenten. |
| Evidence | Ein Nachweis oder eine Nachweisreferenz. Im oeffentlichen Repo stehen nur Metadaten, keine echten Mandatsdokumente. |
| Readiness | Eine Bereitschaftspruefung. Sie sagt, ob ein lokaler Arbeitsplatz oder Plugin-Pfad technisch vorbereitet ist. |
| Trockenlauf | Eine Planvorschau ohne produktive Schreibaktion und ohne echte Mandatsdaten. |

## Merksatz

NoC macht Regeln, offene Fragen und Freigaben sichtbar. Es ist nicht der Ort fuer
echte Akteninhalte.
